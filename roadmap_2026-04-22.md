## FASE 2 — Criterio cuantitativo de degradación de clusters

### Evaluación de 5 métricas

| Métrica | Factibilidad | Ruido | Latencia | Costo cálculo | Recomendación |
|---------|--------------|-------|----------|---------------|---------------|
| **A** Auto-persistencia P(cluster[t+1]=cluster[t]) | ALTA — parsear SIGNALS_RAW | medio | rápida (diaria) | 10-30 min/día | CONFIRMACIÓN |
| **B** Divergencia KL distribución vs training | ALTA — training en regime_wf/JSONs | bajo | lenta (semanal) | 1-5 min/sem | **PRIMARIA** |
| **C** % bars con max_prob<0.5 | MEDIA — cluster_probs no siempre logueado | alto | rápida | 10-30 min/día | DESCARTAR (ruidoso + indirecto) |
| **D** Match rate audit v5.1 | ALTA pero SLOW | bajo | muy lenta (cada N≥50) | 30 min-1h/audit | SECUNDARIA (operacional) |
| **E** Alpha residual sistemático | ALTA — analyzer produce por trade | alto | media (50 trades) | ya calculado | DESCARTAR (confunde factores) |

### Métrica PRIMARIA — B (KL divergence vs training)

**Justificación:** mide directamente si el régimen de mercado ha cambiado respecto al período de training (el GMM "no vio" el régimen actual). Integra sobre tiempo → bajo ruido. Usa data que ya está en producción (SIGNALS_RAW + JSONs expected distribution).

**Cálculo:**

```
Para cada símbolo s:
  Leer regime_wf/sUSDT_specialist_configs.json → expected_dist[s] = {C0: p0, C1: p1, C2: p2}
    (derivable de cluster metadata del walk-forward: n_train_bars por cluster)
  Contar cluster_id en SIGNALS_RAW[s] últimos 30 días (720 cycles)
  observed_dist[s] = {C0: count0/N, C1: count1/N, C2: count2/N}
  KL(observed || expected) = Σ observed_i × log(observed_i / expected_i)
```

**Umbrales sugeridos (con incertidumbre explícita):**

- KL < 0.1 → sano (dist similar a training).
- 0.1 ≤ KL < 0.3 → drift moderado, observar.
- 0.3 ≤ KL < 0.5 → drift significativo, flag.
- KL ≥ 0.5 → **degradación confirmada**, trigger reciclaje si persiste 5 días.

**Ventana:** 30 días rolling (720 cycles) por símbolo. Evaluación semanal.

**Agregación cross-símbolo:** promedio ponderado por n_trades o fraction_of_portfolio. Disparo a nivel portfolio si KL_promedio ≥ 0.3 sostenido 2 semanas, o si ≥5 símbolos individuales cruzan KL ≥ 0.5.

**Limitación:** `expected_dist` exacto puede no estar en los JSONs actuales directamente (requiere extraer de `cluster_metadata` o `period_stats` del walk-forward). Verificar infra disponibility (ver preguntas abiertas).

### Métrica CONFIRMACIÓN — A (Auto-persistencia rolling)

**Justificación:** training esperaba ~97.9% de auto-persistencia (histéresis del GMM + régimen estable). Caída abrupta indica mercado entrando en régimen inestable o GMM clasificando mal.

**Cálculo:**

```
Para cada símbolo s:
  Extraer secuencia cluster_id[t] de SIGNALS_RAW últimos 7 días (168 cycles)
  transitions = count(cluster[t+1] != cluster[t]) / (N-1)
  persistence[s] = 1 - transitions
```

**Umbrales sugeridos:**

- persistence ≥ 0.95 → sano.
- 0.90 ≤ persistence < 0.95 → observar.
- persistence < 0.90 sostenido 7 días → confirmar degradación (en conjunto con B).

**Ventana:** 7 días rolling. Evaluación diaria.

**Falsos positivos esperados:** MEDIOS. Régimen cambiante legítimo puede causar transiciones sin implicar "GMM no sirve". Por eso es confirmación, no primaria.

### Métrica SECUNDARIA — D (Match rate audit)

**Justificación:** cierre de loop. Si B+A señalan degradación estructural, el match rate debería caer (GMM live clasifica distinto que kernel post-hoc sobre data actual porque ambos están siendo "sorprendidos" por régimen nuevo; PERO si solo B+A señalan sin caer match rate, podría ser regime shift legítimo que GMM captura consistentemente en live y post-hoc).

**Uso:** post-audit definitivo N≥50 como punto anchor. Siguientes audits cada ~2 semanas. Si match rate cae >10pp respecto baseline (o cae CI95 fuera del baseline inicial), confirma que degradación afecta decisiones, no solo clasificación.

### Pseudocódigo del detector

```python
def detect_cluster_degradation(
    symbols: list[str],
    cycles_window_days: int = 30,
    min_degradation_days: int = 5,
) -> tuple[bool, dict]:
    """
    Retorna (should_recycle, diagnostics).
    Lógica: B primaria flag sostenida N≥5 días AND A confirmación simultánea → reciclaje.
    B flagged pero A no → observar, no disparar.
    """
    kl_by_symbol = {}
    persistence_by_symbol = {}
    for s in symbols:
        raw = load_signals_raw(s, days=cycles_window_days)
        expected = load_expected_distribution(s)  # from regime_wf JSONs
        kl_by_symbol[s] = kl_divergence(
            observed=count_cluster_frequency(raw),
            expected=expected,
        )
        persistence_by_symbol[s] = auto_persistence(raw[-168:])  # last 7d

    # primary: KL ≥ 0.5 symbol-level OR KL_weighted ≥ 0.3 portfolio-level
    symbols_flagged_primary = [s for s, kl in kl_by_symbol.items() if kl >= 0.5]
    kl_weighted = weighted_mean(kl_by_symbol, weights=trade_count_per_symbol)
    primary_flagged = len(symbols_flagged_primary) >= 5 or kl_weighted >= 0.3

    # confirmation: auto-persistence < 0.90 en ≥50% de los símbolos flagged primary
    if primary_flagged:
        confirmed = sum(
            1 for s in symbols_flagged_primary
            if persistence_by_symbol[s] < 0.90
        ) >= len(symbols_flagged_primary) * 0.5
    else:
        confirmed = False

    # persistence over time: requires state tracking across daily runs
    persistent = check_days_flagged_in_row(
        primary_flagged and confirmed,
        min_days=min_degradation_days,
    )

    should_recycle = persistent
    diagnostics = {
        "kl_by_symbol": kl_by_symbol,
        "persistence_by_symbol": persistence_by_symbol,
        "kl_weighted_portfolio": kl_weighted,
        "symbols_flagged_primary": symbols_flagged_primary,
        "confirmed": confirmed,
        "days_in_row_flagged": ...,
    }
    return should_recycle, diagnostics


def confirm_with_audit_match_rate(baseline_match_rate: float,
                                  current_audit_result: AuditReport) -> bool:
    """Soft secundaria. Llamada post-audit (cada ~2 semanas con N≥50)."""
    return current_audit_result.match_rate < baseline_match_rate - 0.10
```

**Frecuencia evaluación:**

- Diaria (al cierre UTC 00:00, como health_monitor): cálculo de A+B.
- Semanal: reporte consolidado con tendencia, escrito a Telegram.
- Post-audit (cada ~2 semanas): comparación match rate D con baseline.

**Disparo reciclaje:** `should_recycle=True` sostenido ≥5 días consecutivos. NO reciclar por un día aislado de flag (falsos positivos de eventos de mercado transientes).

**Falsos positivos esperados del sistema combinado:** bajos. Requiere AMBAS (B primaria + A confirmación) durante ventana sostenida. El único escenario de falso positivo plausible es un régimen transiente extremo (ej. flash crash en todo el portfolio simultáneamente) que luego revierte — la ventana 5 días lo filtra.
